<objective>
Add a database-backed cache that maps Microsoft 365 SKU GUIDs (`6fd2c87f-b296-...`)
to friendly names (`ENTERPRISEPACK` / `Microsoft 365 E5`). The cache is refreshed
once per 24 hours from `/subscribedSkus` (Graph) and reuses `ExternalServiceData`
(no new model). It is the upstream dependency for the M365 license list rendered
in Plan 03 — without it the UI would show raw GUIDs.

Purpose: Per D-04, license display needs friendly names; per D-07 we cache for 24h;
per the codebase pattern, the `genesys_cache_db.py` daily-refresh shape is the
exact analog. This plan mirrors that analog without inventing a new pattern.

Output: One new service file, one container registration, one hook into the
existing daily refresh loop. No schema/migration changes.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Tampering | SKU rows persisted to ExternalServiceData | mitigate | Only the background-job path writes; no user input flows in. `update_service_data` is parameterized SQL via SQLAlchemy ORM. |
| T-06-07 | Denial of Service | A long Graph call hangs the refresh loop | mitigate | `_make_request` (graph_service) already enforces request timeout. A SKU refresh failure is caught and logged at the call site (Task 2 step 2c). |
| T-06-08 | Information Disclosure | SKU `raw_data` JSONB stored in DB | accept | SKU catalog is non-sensitive tenant metadata (M365 product names + counts). No PII. |
| T-06-09 | Repudiation | Refresh outcomes not auditable | mitigate | `logger.info(f"SKU catalog refreshed: {len(result)} SKUs")` and `logger.error` on failure — both go to stdout JSON logs (OPS-02 from Phase 1). Permission-missing path logs ERROR with the missing permission name. |
