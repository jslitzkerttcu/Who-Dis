<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WhoDis ↔ Microsoft Graph | Outbound HTTPS calls carry the app-registration access token. Untrusted: Graph response payloads (potential field-shape drift). |
| graph_service ↔ Flask request thread | New methods are invoked from `@require_role("viewer")` HTMX endpoints (Plans 03/04). User input (`user_id` path param) crosses into the URL. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | `get_authentication_methods` (MFA methods include sensitive identifiers like phone numbers) | mitigate | Method only callable behind `@require_role("viewer")` (enforced in Plan 03/04 endpoints). No data logged at INFO; use DEBUG for response content. Audit-log via `audit_service.log_search()` in the calling endpoint. |
| T-06-02 | Tampering | `user_id` path parameter passed into Graph URL | mitigate | Graph rejects malformed IDs with 400/404 — pass through as-is. Do NOT interpolate `user_id` into f-strings without encoding for new SQL/log sinks; the only sink is the URL where Graph performs its own validation. |
| T-06-03 | Denial of Service | `get_subscribed_skus` called on every request | accept | Mitigated by SkuCatalogCache (Plan 02) which caches results for 24h via `ExternalServiceData`. This plan exposes the raw fetch only; it is a caller's responsibility to cache. |
| T-06-04 | Elevation of Privilege | Permission-missing sentinel could be misread as success | mitigate | Sentinel uses an explicit `{"error": "permission_missing"}` dict — callers MUST type-check before iterating. Plan 03/04 acceptance criteria enforce this at the renderer. |
| T-06-05 | Repudiation | Graph fetch failures may go unlogged | mitigate | Module-level dedupe set logs ERROR exactly once per missing permission per process startup (D-06 — gives operators a clear signal in stdout JSON logs). Hard exceptions log with `exc_info=True`. |
</threat_model>
