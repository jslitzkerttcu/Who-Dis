# Phase 6 — UI Design Contract

> Visual and interaction contract for Enriched Profiles & Search Export. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase extends the existing search result card with two collapsible enrichment sections (Microsoft 365, Genesys Cloud) and adds per-profile **Copy to Clipboard** and **Export CSV** affordances. All visual decisions inherit from the existing WhoDis Tailwind system declared in `app/templates/base.html`. No new design system, no new component framework — Phase 6 is a strict extension of patterns already in production.

## Spacing Scale

Inherited from Tailwind defaults (multiples of 4). Used values in this phase:

| Token | Tailwind class | Value | Usage in Phase 6 |
|-------|----------------|-------|-------------------|
| xs | `p-1`, `gap-1` | 4px | Icon-to-label gaps inside chips/badges |
| sm | `p-2`, `gap-2` | 8px | Compact spacing inside collapsed section header rows |
| md | `p-4`, `gap-4` | 16px | Default padding inside section content; gap between key:value rows |
| lg | `p-6`, `gap-6` | 24px | Card padding (matches existing search result card `p-6`) |
| xl | `mb-8`, `space-y-8` | 32px | Gap between profile card and enrichment sections; gap between sections |
| 2xl | `py-12` | 48px | Reserved for empty/error states only — not used in normal flow |
| 3xl | — | 64px | Not used in this phase |

**Exceptions:**
- Touch target minimum on Copy/Export buttons: `py-2 px-4` (8/16) — yields 36px tall control, paired with `min-h-[44px]` only on mobile breakpoint via `sm:min-h-0` rollback. Existing search button (line 22) is `px-6 py-2` and is the sizing precedent — match it for primary actions.
- Section expand/collapse chevron padding: `p-3` (12px) — exception to the 4/8/16/24 sequence to align tap target with FontAwesome icon visual center; matches existing dropdown caret behavior in `base.html:131-136`.

| Accent (10%) — Action | `#f2c655` | `bg-ttcu-yellow` | Copy / Export button background (matches existing "Search" button at `search/index.html:22`) |
| Destructive | `#dc2626` (Tailwind `red-600`) | `text-red-600` / `bg-red-50` border `border-red-400` | Permission-degradation message banner inside a section ("Last sign-in unavailable — missing AuditLog.Read.All") |
| Success indicator | `#059669` (Tailwind `green-600`) | `text-green-600` | MFA "registered" pill, presence "Available" dot |
| Warning indicator | `#d97706` (Tailwind `amber-600`) | `text-amber-600` | "Stale cache" indicator on enrichment sections older than 24h, presence "Away" dot |

**Accent reserved for:**
1. **`ttcu-green`** — Microsoft 365 section identity (header icon, focus ring on M365 controls). Never used as a generic action color.
2. **`genesys-orange`** — Genesys Cloud section identity (header icon, focus ring on Genesys controls). Never used outside Genesys-sourced data.
3. **`ttcu-yellow`** — Primary action affordance on Copy / Export buttons only. Matches existing Search button precedent. Never used for status, never for borders, never on links.

**60/30/10 split:** ~60% gray-50 page surface, ~30% white card surfaces (default card + two enrichment section panels), ~10% accent (chevron tints, section identity icons, action button fills). No new neutral tokens introduced.

**Destructive color is explicitly NOT used for the permission-degradation message body text** — that uses `text-amber-700` on `bg-amber-50` because the user is not blocked, just informed of a missing capability. `text-red-*` is reserved for hard error states (Graph 5xx, network failure on lazy-load).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — Copy | `Copy to Clipboard` (icon: `fa-copy`) |
| Primary CTA — Export | `Export CSV` (icon: `fa-file-csv`) |
| Section header — M365 | `Microsoft 365` (icon: `fa-microsoft`, color `text-ttcu-green`) |
| Section header — Genesys | `Genesys Cloud` (icon: `fa-headset`, color `text-genesys-orange`) |
| Section default state | Collapsed. Header reads `Microsoft 365` / `Genesys Cloud` with right-aligned chevron `fa-chevron-down`. No preview line. |
| Section expanding | Header chevron rotates to `fa-chevron-up`. Body shows skeleton rows (3 lines, 200ms shimmer — reuse `createSkeleton('text')` from `base.html:404`) until HTMX response arrives. |
| Section loaded | Body shows key:value rows. Each value carries an inline source chip e.g. `[Graph]` or `[Genesys]` in `text-xs text-gray-500`. |
| Empty section (no data returned) | `No Microsoft 365 data available for this user.` / `No Genesys Cloud profile for this user.` — `text-sm text-gray-500 italic`, centered, `py-6`. |
| Permission-degradation row (D-06) | `<Field label> unavailable — missing <Permission>.` Example: `Last sign-in unavailable — missing AuditLog.Read.All.` Rendered inline in the section body as `text-amber-700 bg-amber-50 border-l-4 border-amber-400 px-4 py-2 text-sm` — never a toast, never a modal. |
| Error state (HTMX 5xx on lazy-load) | `Couldn't load Microsoft 365 data. Try the Refresh button or contact IT.` — `text-red-700 bg-red-50 border-l-4 border-red-400 px-4 py-2 text-sm`. Includes a `Retry` link that re-fires the HTMX request. |
| Stale cache indicator | `Cached <N>h ago` — `text-xs text-amber-600` next to the section header timestamp. Appears when EmployeeProfile cache age > 20h (approaching the 24h TTL). |
| Copy success toast | `Copied profile to clipboard` — uses existing `showToast(msg, 'success')` from `base.html:256`. Duration 3000ms. |
| Copy failure toast | `Couldn't copy. Select the text and copy manually.` — `showToast(msg, 'error')`. |
| Export filename | `whodis-<sanitized-username>-<YYYYMMDD>.csv` (e.g. `whodis-jdoe-20260426.csv`). |
| CSV "not loaded" cell value | `Not loaded` (literal) — paired with source attribution `[Genesys, not loaded]` in the source column. Not `null`, not empty, not `N/A`. |
| Clipboard plain-text format (D-13) | Key:value lines, one per visible field, in display order. Section breaks separated by a blank line. Example block: `Name: Jane Doe\nTitle: Senior Analyst\nDept: IT\n\nMicrosoft 365\nMFA: registered (Authenticator, SMS)\nLast sign-in: 2026-04-23\nLicenses: Microsoft 365 E5, Power BI Pro\n\nGenesys Cloud\nNot loaded` |
| Destructive confirmation | **None in this phase.** Phase 6 is read-only. Write actions (license assign/remove) are explicitly deferred to Phase 9 per CONTEXT.md `<deferred>`. |

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Section expand (first time) | Click header → HTMX `GET /search/api/profile/<id>/m365` (or `/genesys`) with `hx-swap="innerHTML"` into the section body div. `hx-trigger="click once"` to prevent re-fetch. While in flight, body shows 3 shimmer skeleton rows. Pattern verbatim from `app/blueprints/search/__init__.py:680` (`/api/signin-logs/<id>`). |
| Section collapse | Click header again → CSS-only collapse via `hidden` class toggle on the body div. Loaded data is preserved in DOM (no re-fetch on next expand). |
| Re-expand after collapse | Reveal cached DOM. No network call. |
| Section refresh | Existing per-profile force-refresh button (Phase 1) invalidates EmployeeProfile cache; on next expand, full re-fetch occurs. No new refresh control added at section level in v1. |
| Copy to Clipboard | Click button → JS reads currently-rendered section DOM, formats per D-13 plain-text spec, calls `navigator.clipboard.writeText()`, fires `showToast('Copied profile to clipboard', 'success')`. WYSIWYG — unexpanded sections export as `Not loaded`. No auto-fetch (D-12). |
| Export CSV | Click button → server endpoint `GET /search/api/profile/<id>/export.csv` returns `text/csv` with `Content-Disposition: attachment; filename="..."`. Server reads only what's currently in the EmployeeProfile cache; for unexpanded data the source column reads `[<source>, not loaded]` and the value column reads `Not loaded`. |
| Keyboard | Section header is a `<button>` (not a `<div>`), focusable via Tab, expandable via Space/Enter. Chevron rotation animates 150ms. Focus ring uses section accent color (`focus:ring-ttcu-green` for M365, `focus:ring-genesys-orange` for Genesys). |
| Screen reader | Section header carries `aria-expanded="false|true"` and `aria-controls="<section-body-id>"`. Body div carries `role="region"` and `aria-labelledby="<header-id>"`. `aria-busy="true"` set on body during HTMX in-flight. |
| Mobile (≤768px) | Sections stack full-width. Copy/Export buttons stack vertically under the result card with `w-full sm:w-auto`. Chevron tap target is `min-h-[44px]` on mobile only. |
| HTMX request error | `htmx:responseError` shows the error-state copy from the table above, with `Retry` link that triggers `htmx.trigger(elt, 'click')`. 401 follows existing redirect-to-login flow in `base.html:67-90`. |
