# Phase 9 -- UI Design Contract

> Visual and interaction contract for Write Operations. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Phase 9 adds write-action affordances to the existing search result profile sections (Phase 6 collapsible areas). AD actions (unlock, reset password, enable/disable) appear in the AD/identity section. License actions (assign, remove, swap) appear in the M365 section. All write actions require a confirmation modal with mandatory reason text, produce an audit trail, and report results via the existing toast notification system. This phase introduces one new shared partial (confirmation modal) and one new component (dismissible password banner). All visual decisions inherit from the established WhoDis Tailwind system.

## Color

60/30/10 split inherited from Phase 6 UI-SPEC.

| Role | Value | Tailwind | Usage |
|------|-------|----------|-------|
| Dominant (60%) | `#f9fafb` | `bg-gray-50` | Page background (existing) |
| Secondary (30%) | `#ffffff` | `bg-white` | Modal surface, action button default state |
| Accent (10%) -- AD actions | `#007c59` | `text-ttcu-green` / `bg-ttcu-green` | Unlock and Enable action button fills, success states |
| Accent (10%) -- License actions | `#007c59` | `text-ttcu-green` / `bg-ttcu-green` | Assign License button fill |
| Destructive | `#dc2626` | `text-red-600` / `bg-red-600` | Disable Account button, Remove License button, destructive modal confirm button |
| Warning | `#d97706` | `text-amber-600` / `bg-amber-50` | Risk-tier warning banner inside modal for high-risk actions (disable, license remove) |
| Success indicator | `#059669` | `text-green-600` / `bg-green-50` | Success toast background tint, post-action checkmark icon |
| Error indicator | `#dc2626` | `text-red-600` / `bg-red-50` | Error toast, failed action icon swap |
| Password banner | `#1e40af` | `bg-blue-50 border-blue-400` | Temporary password display banner -- distinct from success/error to avoid confusion |

**Accent reserved for:**
1. **`ttcu-green`** -- Non-destructive write action buttons (Unlock Account, Enable Account, Assign License, Reset Password confirm). Green signals a constructive/enabling action.
2. **`red-600`** -- Destructive write action buttons (Disable Account, Remove License). Red signals irreversible or high-impact action. Used on the modal Confirm button for destructive actions only.
3. **`ttcu-yellow`** -- Not used for write action buttons. Reserved for Copy/Export per Phase 6. Avoids action-color overload.

**Risk-tier visual differentiation (D-14 discretion):**
- **Standard actions** (unlock, reset password, enable, assign license): Modal header uses `text-gray-900`. Confirm button uses `bg-ttcu-green hover:bg-green-700 text-white`.
- **High-risk actions** (disable account, remove license): Modal header uses `text-gray-900`. An amber warning banner appears below the target user echo: `This action will immediately affect the user's access.` Confirm button uses `bg-red-600 hover:bg-red-700 text-white`.
- **License swap**: Uses standard-action styling (green confirm) because the intent is constructive (upgrade/change), with the amber warning only if the remove step could leave the user without any license.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| **Action button click** | Opens confirmation modal as a fixed overlay (`fixed inset-0 bg-gray-600 bg-opacity-50 z-50`). Modal centers vertically at `top-20` on desktop. Matches existing pattern from `admin/_cache_actions.html:163`. |
| **Reason field validation** | Confirm button starts disabled (`opacity-50 cursor-not-allowed`). Enables when reason textarea has 3+ non-whitespace characters. Real-time validation via `input` event listener. |
| **Confirm action** | POST via HTMX to `/search/api/write/{action}` with `hx-swap="none"` (no DOM swap -- response triggers toast via `HX-Trigger` response header). Confirm button shows spinner, disables. Modal closes on success response. On error, modal stays open with error message below the form. |
| **Cancel / Escape / Click-outside** | Closes modal, clears reason field. No network call. |
| **Password banner (post-reset)** | Appears as a fixed-position banner below the nav bar (`fixed top-16 left-0 right-0 z-40`). Blue-tinted (`bg-blue-50 border-b-2 border-blue-400`). Contains the generated password in a monospace font (`font-mono text-lg`), a show/hide toggle (`fa-eye` / `fa-eye-slash`), a copy button (`fa-copy`), and a dismiss link. Password initially visible (admin needs to read it immediately). |
| **Password show/hide** | Toggle between `type="text"` and masked dots display. Icon swaps between `fa-eye` and `fa-eye-slash`. |
| **Password copy** | Click copy button -> `navigator.clipboard.writeText(password)` -> `showToast('Password copied to clipboard', 'success', 3000)`. |
| **Password banner dismiss** | Click Dismiss -> banner slides up and removes from DOM. Cannot be recovered -- password is gone. The "will not be shown again" warning is always visible. |
| **Post-action button state** | After successful action, the action button briefly (2s) swaps its icon to `fa-check` with `text-green-600`, then reverts. For enable/disable toggle, the button swaps to the opposite state (Enable becomes Disable and vice versa). For unlock, button becomes disabled with tooltip "Account is not locked". |
| **License assign modal** | Extends the standard modal with a dropdown (`<select>`) of available SKUs from the organization's subscribed licenses (fetched via HTMX on modal open). Dropdown shows `{displayName} ({available}/{total} available)`. |
| **License remove modal** | Standard confirmation modal. Target license name shown in the target echo: `Remove: {license_displayName} from {user_name}`. |
| **License swap modal** | Extended modal (`max-w-lg`). Two dropdowns: "Remove" (pre-populated with user's current licenses) and "Assign" (available SKUs). Both required. Confirm button label: `Confirm Swap`. |
| **Action button visibility (D-12)** | Action buttons are hidden for non-admin users (not rendered in the template). For a 4-5 person team where everyone knows their role, hidden-not-disabled avoids confusion. Server-side `{% if g.role == 'admin' %}` gate. |
| **License action placement (D-13)** | License actions are inline: each license chip in the M365 section gets a small `fa-times` remove icon on hover (admin only). "Assign License" and "Swap License" buttons appear below the license list as a button group. This keeps actions contextual without a separate sub-panel. |
| **Keyboard** | Modal traps focus. Tab cycles through reason textarea and buttons. Escape closes modal. Confirm button is not in tab order while disabled. |
| **Screen reader** | Modal has `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to modal title. Reason textarea has `aria-required="true"`. Disabled buttons have `aria-disabled="true"` with `aria-label` explaining why. |
| **Mobile (<=768px)** | Modal goes full-width with `mx-4` margin. Action buttons in the profile section stack vertically with `w-full`. Password banner is full-width fixed. |
| **HTMX error handling** | Non-2xx response from write endpoint: modal stays open, error message appears below the form in `text-red-600 text-sm`. 401 follows existing redirect-to-login flow. 429 shows `Rate limit exceeded. Please wait and try again.` |
