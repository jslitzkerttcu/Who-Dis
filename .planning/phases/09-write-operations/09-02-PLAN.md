    Pattern: Follow api_tokens.py structure. Each endpoint:
    - Decorator stack: @search_bp.route("/api/write/{action}", methods=["POST"]) then @require_role("admin") then @csrf_double_submit.protect then @handle_errors(json_response=True)
    - Import from app.middleware.auth import require_role; from app.middleware.csrf import csrf_double_submit; from app.utils.error_handler import handle_errors
    - Get write_operations service via current_app.container.get("write_operations")
    - Read form data: user_dn, display_name, reason from request.form
    - Validate reason: strip whitespace, require >= 3 chars, return 400 if invalid
    - Call appropriate WriteOperationsService method
    - On success: return make_response("", 200) with HX-Trigger header containing json.dumps({"showToast": {"message": "...", "type": "success"}})
    - On failure: return error message as text with 500 status

    Four AD endpoints:
    1. POST /search/api/write/unlock -- calls write_ops.unlock_account(user_dn, display_name, reason). Success toast: "Account unlocked successfully"
    2. POST /search/api/write/reset-password -- calls write_ops.reset_password(user_dn, display_name, reason). On success returns render_template("search/_password_banner.html", password=result["data"]["password"]) with 200 status. HX-Trigger toast: "Password reset successfully"
    3. POST /search/api/write/enable -- calls write_ops.set_account_enabled(user_dn, display_name, True, reason). Success toast: "Account enabled successfully"
    4. POST /search/api/write/disable -- calls write_ops.set_account_enabled(user_dn, display_name, False, reason). Success toast: "Account disabled successfully"

    For reset-password endpoint: the response body IS the password banner HTML fragment (not empty). The HX-Trigger header fires the toast. The caller uses hx-target to place the banner.
