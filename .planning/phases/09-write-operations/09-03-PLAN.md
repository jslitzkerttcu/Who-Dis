    **License endpoints** -- Add to app/blueprints/search/write_routes.py (extending the file from Plan 02):

    1. GET /search/api/write/available-skus -- Returns _license_select.html fragment. Fetches org SKUs via graph_service.get_subscribed_skus(). For each SKU compute available count (prepaidUnits.enabled - consumedUnits). Build list of {sku_id, display_name (from sku_catalog.get_sku_name or skuPartNumber fallback), available, total}. Render template. Accepts optional query param exclude_sku_id to filter out a specific SKU (used in swap modal). Requires @require_role("admin") but no CSRF (GET request).

    2. POST /search/api/write/assign-license -- Read form fields: user_id, user_email, sku_id, sku_name, reason. Validate reason >= 3 chars. Call write_ops.assign_license(user_id, user_email, sku_id, sku_name, reason). On success return empty 200 with HX-Trigger showToast "License assigned successfully". On failure return 500 with error message. Check for permission_missing sentinel in result and return specific "Graph API permission missing" error.

    3. POST /search/api/write/remove-license -- Read form: user_id, user_email, sku_id, sku_name, reason. Same validation. Call write_ops.remove_license(). Toast "License removed successfully". This is a high-risk action (per D-14).

    4. POST /search/api/write/swap-license -- Read form: user_id, user_email, old_sku_id, old_sku_name, new_sku_id, new_sku_name, reason. Call write_ops.swap_license(). Handle result dict:
       - success=True: toast "License swapped successfully"
       - success=False, rollback_needed=True, rollback_success=True: toast (warning) "License swap incomplete but rollback succeeded. {old_sku_name} has been restored."
       - success=False, rollback_needed=True, rollback_success=False: return 500 with HX-Trigger containing showBanner event (not showToast) with type "error" and message "CRITICAL: {user_email} may have no {old_sku_name} or {new_sku_name} assigned. Manual intervention required. Check audit log." Per D-09 this is a persistent error banner.
       - success=False, rollback_needed=False: standard error toast

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> License endpoints | Form data (user_id, sku_id, reason) crosses trust boundary |
| App -> Graph API | License mutations require LicenseAssignment.ReadWrite.All permission |
| SKU dropdown -> Client | Available SKU list exposed to admin users only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-14 | Elevation of Privilege | License endpoints | mitigate | @require_role("admin") on all license endpoints; server-side role check |
| T-09-15 | Tampering | sku_id parameter | mitigate | sku_id validated as non-empty string; Graph API itself validates SKU existence. Invalid SKU returns Graph 400 error, surfaced as user-visible error. |
| T-09-16 | Denial of Service | Swap partial failure | mitigate | Rollback attempted; double failure logged at ERROR with MANUAL_INTERVENTION_REQUIRED; admin sees persistent banner per D-09 |
| T-09-17 | Information Disclosure | Available SKUs | accept | SKU availability (count) is organizational metadata, acceptable for admin users to see |
| T-09-18 | Tampering | CSRF on license POST | mitigate | @csrf_double_submit.protect on all POST endpoints |
| T-09-19 | Spoofing | user_id in form data | mitigate | user_id originates from server-rendered profile section (from prior Graph API search); audit trail records both operator and target |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
