    1. `unlock_account(self, user_dn: str) -> bool`: Create Server+Connection (same pattern as test_connection at line 69). Call conn.modify(user_dn, {'lockoutTime': [(MODIFY_REPLACE, ['0'])]}). Return bool result. Catch LDAPException, log error, return False. Add MODIFY_REPLACE to the ldap3 import at line 4.

    2. `reset_password(self, user_dn: str, new_password: str) -> bool`: Check self.use_ssl first -- if False, log error "Password reset requires SSL/TLS connection" and return False (per Pitfall 1 from RESEARCH.md). Create Server+Connection. Call conn.extend.microsoft.modify_password(user_dn, new_password). Return bool result. Never log the password value.

    3. `set_account_enabled(self, user_dn: str, enabled: bool) -> bool`: Create Server+Connection. Search for current userAccountControl value using conn.search(user_dn, '(objectClass=*)', search_scope='BASE', attributes=['userAccountControl']). Read current UAC as int. If enabled: new_uac = current_uac & ~2 (clear ACCOUNTDISABLE bit). If disabled: new_uac = current_uac | 2 (set ACCOUNTDISABLE bit). Call conn.modify with MODIFY_REPLACE. Return bool. Per Pitfall 2: never overwrite the full UAC -- only toggle bit 1.

    **Graph license methods** -- Add three methods to GraphService class in app/services/graph_service.py:

    1. `assign_license(self, user_id: str, sku_id: str, disabled_plans: list = None) -> dict`: Get token via self.get_access_token(). POST to {graph_base_url}/users/{user_id}/assignLicense with body {"addLicenses": [{"skuId": sku_id, "disabledPlans": disabled_plans or []}], "removeLicenses": []}. On 403 return _permission_missing("LicenseAssignment.ReadWrite.All"). Return {"success": True} on 200, {"success": False, "error": str} otherwise.

    2. `remove_license(self, user_id: str, sku_id: str) -> dict`: Same pattern, body has {"addLicenses": [], "removeLicenses": [sku_id]}. Same error handling.

    3. `swap_license(self, user_id: str, old_sku_id: str, new_sku_id: str) -> dict`: First attempt atomic swap -- single POST with addLicenses=[{skuId: new_sku_id}] AND removeLicenses=[old_sku_id] (per D-08 / Don't Hand-Roll from RESEARCH). If atomic succeeds return {"success": True}. If atomic fails (HTTPError), fall back to two-call: call remove_license then assign_license. If assign fails after remove succeeds, attempt rollback by calling assign_license(user_id, old_sku_id). Return dict with keys: success, rollback_needed (bool), rollback_success (bool or None), error (str or None). Per D-09: log full state for manual recovery on double failure.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App -> LDAP Server | Write operations cross to external AD; bind credentials used |
| App -> Graph API | License mutations require OAuth bearer token with write permissions |
| Coordinator -> Audit DB | Every write action must create audit record regardless of outcome |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | reset_password | mitigate | Never log generated password; audit details dict excludes password; password exists only in return dict |
| T-09-02 | Elevation of Privilege | write_operations.py | mitigate | Coordinator does not enforce auth -- callers (route handlers in Plan 02/03) must use @require_role("admin") |
| T-09-03 | Tampering | LDAP modify | mitigate | user_dn validated by prior LDAP search (not user-supplied free text); UAC bit manipulation preserves other flags |
| T-09-04 | Denial of Service | Graph license swap | mitigate | Rollback attempt on partial failure; ERROR log with MANUAL_INTERVENTION_REQUIRED on double failure |
| T-09-05 | Repudiation | All write actions | mitigate | audit_logger.log_admin_action called for every action with who/what/target/when/ip/reason per WRIT-05 |
| T-09-06 | Information Disclosure | LDAP bind credentials | accept | Credentials read from encrypted config (existing pattern); no new exposure surface |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan; all operations use existing ldap3, requests, msal |
</threat_model>
