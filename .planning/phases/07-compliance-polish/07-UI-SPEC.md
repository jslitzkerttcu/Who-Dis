## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps (mr-1), badge internal padding (px-2.5 py-0.5) |
| sm | 8px | Compact element spacing (space-x-2, gap-2), table cell vertical (py-2) |
| md | 16px | Default element spacing (gap-4, p-4), card internal padding |
| lg | 24px | Section padding (p-6), card body padding, table cell horizontal (px-6) |
| xl | 32px | Layout gaps between major cards (gap-8) |
| 2xl | 48px | Progress bar section vertical spacing (py-12 loading states) |
| 3xl | 64px | Not used this phase |

Exceptions: Progress bar height is 8px (h-2 Tailwind class). Sort header click targets maintain minimum 32px height for accessibility.

## Component Inventory

### New Components (this phase adds)

| Component | Type | Location |
|-----------|------|----------|
| Progress Bar | HTMX partial | `admin/partials/_compliance_progress.html` |
| Sortable Table Header | JS-enhanced Jinja | `admin/partials/_compliance_violations_table.html` (enhance existing) |
| Sync Status Card | HTMX partial | `admin/partials/_warehouse_sync_status.html` |
| CSV Export Button | Button (conditionally visible) | Inline in violations table wrapper |

### Progress Bar Specification

```
Container: w-full bg-gray-200 rounded-full h-2 overflow-hidden
Fill: bg-blue-600 rounded-full h-2 transition-all duration-300
Text below: text-sm text-gray-600 mt-2
Layout: flex justify-between items-center (counter left, percentage right)
```

States:
- **Idle:** Hidden (display:none)
- **Running:** Visible, bar fills proportionally, counter updates every 2s via HTMX poll
- **Complete:** Bar reaches 100%, auto-swaps to results table via hx-swap="outerHTML"
- **Error:** Bar stops, red text appears below: "{error_count} errors encountered"

HTMX behavior:
- `hx-get="/api/admin/jobs/compliance_check/status/{run_id}"` 
- `hx-trigger="every 2s"`
- `hx-swap="outerHTML"` (server controls when to swap progress for results)

### Sortable Table Headers Specification

```
Header cell: cursor-pointer select-none hover:bg-gray-100 transition-colors
Active sort: text-gray-900 (instead of text-gray-500)
Icon: ml-1 inline text-xs
```

Behavior:
- Click toggles: neutral -> ascending -> descending -> ascending (no neutral reset)
- Default sort on page load: severity descending (critical first)
- Sort is client-side JavaScript (no server round-trip per D-05)
- Sort rank map: `{critical: 4, high: 3, medium: 2, low: 1}`

### Sync Status Card Specification

```
Container: bg-white rounded-lg shadow-sm border border-gray-200 p-4
Layout: flex items-center justify-between
Left: icon + "Last synced: {relative_time}" or error message
Right: "Sync Now" button (inline-flex items-center px-3 py-2 border...)
```

States:
- **Success:** Green check icon + relative timestamp ("Last synced: 2 hours ago")
- **Error:** Red warning icon + categorized error message (from D-09)
- **Syncing:** Spinning icon + "Syncing..." + button disabled/grayed
- **Never synced:** Gray clock icon + "Never synced"

### CSV Export Button Specification

```
Position: Above violations table, right-aligned in a flex justify-between toolbar
Visibility: Hidden when no completed run exists; visible after run completes
Style: Same as existing secondary button pattern (border border-gray-300, text-gray-700, bg-white hover:bg-gray-50)
Icon: fas fa-download mr-2
```

Behavior:
- Triggers `GET /admin/api/compliance-export/{run_id}` with `Content-Disposition: attachment`
- Always exports full run (ignores client-side sort/filter state per D-12)
- CSV includes metadata header rows (D-13) then data columns (D-14)

## Interaction States

### Run Compliance Check Button

| State | Appearance | Behavior |
|-------|------------|----------|
| Idle | Blue-600 bg, white text, play icon | Click fires POST to job endpoint |
| Running | Disabled, spinner icon, "Running..." text | Prevented from re-triggering (mutex) |
| Complete | Returns to idle | Dashboard refreshes automatically |

### Sync Now Button

| State | Appearance | Behavior |
|-------|------------|----------|
| Idle | Secondary style (white bg, gray border) | Click fires POST to sync job endpoint |
| Syncing | Disabled, opacity-50, spinner, "Syncing..." | Prevented from re-triggering |
| Complete | Returns to idle, status card updates | Timestamp refreshes |
| Error | Returns to idle, error state shown in status card | Error message displayed |
