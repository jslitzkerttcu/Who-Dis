| Boundary | Description |
|----------|-------------|
| Portal -> Jobs API | External SandCastle scheduler sends M2M bearer token |
| Admin browser -> Jobs API | Authenticated admin session via Keycloak OIDC |
| Jobs API -> Background thread | Job runner executes with app context, no user session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | Spoofing | portal_auth.py | mitigate | RS256 signature verification via Keycloak JWKS; azp claim must equal "sandcastle-scheduler" |
| T-07-05 | Elevation of Privilege | /api/admin/jobs POST | mitigate | admin_or_portal_required decorator on all routes; no anonymous access |
| T-07-06 | Information Disclosure | /api/admin/jobs/status | mitigate | Run IDs are UUID4 (unguessable); endpoint requires admin auth |
| T-07-07 | Denial of Service | trigger_job endpoint | mitigate | Mutex + 409 Conflict prevents concurrent runs; single worker caps threads |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed — PyJWT already in requirements.txt |
</threat_model>
