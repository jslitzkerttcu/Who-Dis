  <action>
    Rewrite Dockerfile as a two-stage multi-stage build per D-05 and RESEARCH.md Pattern 3.

    STAGE 1 (builder) -- FROM python:3.12-slim AS builder:
    1. apt-get install gnupg2, curl, unixodbc-dev (build-only tools)
    2. Import Microsoft GPG key and add msodbcsql18 apt repo (same commands as current Dockerfile)
    3. ACCEPT_EULA=Y apt-get install msodbcsql18
    4. Clean apt lists
    5. COPY requirements.txt and pip install --no-cache-dir --prefix=/install -r requirements.txt

    STAGE 2 (runtime) -- FROM python:3.12-slim:
    1. Create non-root user: groupadd -r app && useradd -r -g app -u 10001 app (same as current)
    2. WORKDIR /app
    3. apt-get install only runtime deps: unixodbc, postgresql-client (NO gnupg2, NO curl, NO unixodbc-dev)
    4. Clean apt lists
    5. COPY --from=builder /opt/microsoft /opt/microsoft (ODBC driver binaries)
    6. COPY --from=builder /etc/odbcinst.ini /etc/odbcinst.ini (ODBC driver registration)
    7. COPY --from=builder /install /usr/local (pip packages)
    8. COPY requirements.txt . (for reference/rebuild detection -- layer cache optimization per DEVOPS-03)
    9. COPY . . (source code -- last to maximize cache hits)
    10. COPY scripts/docker_healthcheck.py /usr/local/bin/healthcheck.py
    11. RUN chmod +x ./docker-entrypoint.sh && chown -R app:app /app
    12. USER app
    13. EXPOSE 5000
    14. HEALTHCHECK --interval=30s --timeout=10s --start-period=20s --retries=3 CMD python /usr/local/bin/healthcheck.py || exit 1
    15. Preserve all existing ENV variables from current Dockerfile
    16. ENTRYPOINT ["./docker-entrypoint.sh"]

    Per RESEARCH.md Pitfall 1: copy BOTH /opt/microsoft/ AND /etc/odbcinst.ini from builder.
    Per RESEARCH.md Pitfall 5: keep postgresql-client in runtime for pg_isready in docker-entrypoint.sh.

    Per Claude's discretion on ODBC driver installation and runtime contents.
