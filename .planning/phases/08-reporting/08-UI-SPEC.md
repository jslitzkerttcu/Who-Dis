## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`) | Icon gaps, inline padding |
| sm | 8px (`p-2`, `gap-2`) | Compact element spacing, button icon gaps, filter button vertical padding |
| md | 16px (`p-4`, `gap-4`, `px-4`) | Default card padding, element spacing |
| lg | 24px (`p-6`, `gap-6`, `py-6`) | Card inner padding, section gaps |
| xl | 32px (`gap-8`, `mb-8`) | Layout column gaps, major section breaks |
| 2xl | 48px (`py-12`) | Not used this phase |
| 3xl | 64px (`py-16`) | Not used this phase |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (normal) | 1.5 | `text-sm font-normal` |
| Label | 14px | 400 (normal) | 1.4 | `text-sm font-normal text-gray-500 uppercase tracking-wider` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` |
| Page Title | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold` |

Weights used: 2 (400 normal, 600 semibold). Labels are distinguished from body text by uppercase letterform and muted color (`text-gray-500`), not by weight. Page titles are distinguished from headings by size (24px vs 20px), not by weight.

Source: Derived from existing admin templates (`compliance_dashboard.html`, `admin/index.html`). Collapsed to 2-weight system for visual consistency.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-gray-50` (#F9FAFB) | Page background |
| Secondary (30%) | `bg-white` (#FFFFFF) | Cards, tables, panels |
| Accent (10%) | `bg-blue-600` (#2563EB) | Primary action buttons, active tab indicator, focus rings, progress spinners |
| Destructive | `text-red-600` / `bg-red-100` (#DC2626 / #FEE2E2) | Error states, failed sync indicators, critical severity badges |

### Semantic Colors (report-specific)

| Color | Class | Reserved For |
|-------|-------|-------------|
| Green | `text-green-600` / `bg-green-100` | Success states: sync complete, healthy metrics, high MFA adoption |
| Yellow/Amber | `text-yellow-600` / `bg-yellow-100` | Warning states: stale cache, medium severity, degraded metrics |
| Red | `text-red-600` / `bg-red-100` | Error states: failed sync, low adoption, critical violations |
| Gray | `text-gray-500` / `bg-gray-100` | Neutral states: no data yet, informational badges |
| TTCU Green | `bg-ttcu-green` (#007c59) | Brand accent on admin card icons (existing pattern) |
| Genesys Orange | `text-genesys-orange` (#FF4F1F) | Genesys-specific section headers and status indicators |

Accent reserved for: primary CTA buttons ("Export CSV", "Refresh Data"), active report tab underline, focus rings on interactive elements, loading spinners. Never used for decorative elements or static text.

## Component Inventory

### Report Tab Navigation

Structure: Horizontal tab bar at the top of the reports page. Each tab corresponds to a report type.

| Tab | Icon | Label |
|-----|------|-------|
| 1 | `fa-id-card-clip` | Licenses |
| 2 | `fa-shield-halved` | Security |
| 3 | `fa-headset` | Contact Center |
| 4 | `fa-clock-rotate-left` | Run History |

Active tab: `border-b-2 border-blue-600 text-blue-600 font-semibold`
Inactive tab: `text-gray-500 hover:text-gray-700 hover:border-gray-300`
Tab container: `border-b border-gray-200`

Implementation: Standard anchor-based tabs with HTMX `hx-get` loading tab content as HTML fragments. Each tab is a separate server route returning a partial template. Use `hx-push-url` so tab state is bookmarkable.

Source: D-01 (Claude's discretion) — tabs chosen over sidebar sub-pages to match existing admin patterns (compliance dashboard uses flat page layout, not sidebar).

### Genesys Agent Status Section (REPT-05)

Layout: List or table of Genesys-enabled users with presence indicators.
Loading: HTMX lazy-load on tab activation (`hx-trigger="revealed"`) — matches Phase 6 pattern (D-09).

Columns: Agent Name | Presence | Routing Status | Queues
Presence indicator: Colored dot before status text.
- On Queue: `bg-green-500` dot + "On Queue"
- Available: `bg-green-400` dot + "Available"  
- Away/Break: `bg-yellow-400` dot + "Away"
- Busy: `bg-red-400` dot + "Busy"
- Offline: `bg-gray-400` dot + "Offline"

No pagination (small agent population per CLAUDE.md: ~4-5 IT users, but full Genesys roster may be larger — paginate if > 50 agents).

### Stale-Cache Indicator (REPT-08)

Design: Inline timestamp badge in the tab header area, right-aligned next to the CSV export button.

```html
<div class="flex items-center text-sm">
    <!-- Fresh -->
    <i class="fas fa-clock text-green-500 mr-2"></i>
    <span class="text-gray-500">Updated <time title="2026-05-17 14:30:00">12 min ago</time></span>
    
    <!-- Stale -->
    <i class="fas fa-clock text-yellow-500 mr-2"></i>
    <span class="text-yellow-600 font-semibold">Data is 5h old</span>
    <button class="ml-2 text-blue-600 hover:text-blue-800 text-sm font-semibold">Refresh Data</button>
</div>
```

Fresh threshold: Within TTL (4h licenses, 1h security, 5min Genesys).
Stale threshold: Beyond TTL — icon turns yellow, text becomes semibold, manual refresh button appears.

Source: D-11 (Claude's discretion) — timestamp+badge chosen as simplest effective approach, matching the `_warehouse_sync_status.html` pattern.

## Interaction Contracts

### Tab Switching

| Trigger | Action | Target |
|---------|--------|--------|
| Click tab | `hx-get="/admin/reports/{tab}"` | `#report-content` (main content area below tabs) |
| URL param | `?tab=licenses` | Server renders correct tab as active on page load |
| `hx-push-url` | Updates browser URL to `/admin/reports?tab={name}` | Bookmarkable tab state |

### Date Range Filter (Failed Sign-ins)

| Trigger | Action | Target |
|---------|--------|--------|
| Click preset button (24h/7d/30d) | `hx-get` with `?window=24h` | `#signin-table` |
| Click "Apply Filter" (custom range) | `hx-get` with `?from=...&to=...` | `#signin-table` |
| Active preset | Gets `bg-blue-600 text-white` class; others revert to gray | Visual feedback only |

### Lazy-Load Genesys Tab

| Trigger | Action | Target |
|---------|--------|--------|
| Tab revealed | `hx-trigger="revealed"` fires `hx-get="/admin/reports/genesys"` | `#genesys-content` |
| Loading state | Spinner + "Loading agent status..." text | Inline in tab content area |

### CSV Export

| Trigger | Action | Response |
|---------|--------|----------|
| Click "Export CSV" | `GET /admin/reports/{type}/export?{current_filters}` | Browser download (Content-Disposition: attachment) |
| Includes current filters | Date range for sign-ins, all data for licenses/security | CSV with metadata header rows |

### Stale-Cache Refresh

| Trigger | Action | Target |
|---------|--------|--------|
| Click "Refresh Data" on stale indicator | `hx-post` to trigger report sync job | Replace indicator with spinner |
| Job completes | Indicator updates to fresh timestamp | Auto via HTMX polling or swap |

### Table Sorting (Client-Side)

| Trigger | Action |
|---------|--------|
| Click column header | JS sorts table rows in memory (ascending/descending toggle) |
| Sort indicator | `fa-sort-up` / `fa-sort-down` icon appended to active column header |

Reuse: Phase 7 `compliance-sort.js` pattern.
