- id (Integer, primary_key, autoincrement) — inherited from BaseModel

- store(report_type, cache_key, data, ttl_hours) -> ReportCache: upsert pattern — query by (report_type, cache_key), update data+generated_at if exists, else insert. Commit. Return instance.
- get_cached(report_type, cache_key) -> Optional[ReportCache]: simple query by type+key.

Follow the import pattern from sync_metadata.py: from app.database import db, from app.models.base import BaseModel, TimestampMixin.

**GraphService additions** (app/services/graph_service.py) — add three new methods:

1. get_all_users_with_licenses(self) -> List[Dict[str, Any]]:
   Paginated iteration over /beta/users with $select=displayName,userPrincipalName,assignedLicenses,signInActivity, $top=500. Add ConsistencyLevel: eventual header. Follow @odata.nextLink until exhausted. Handle 403 with _permission_missing("User.Read.All"). Log total user count on completion. Per D-05.

2. get_mfa_registration_details(self) -> List[Dict[str, Any]]:
   Paginated iteration over https://graph.microsoft.com/v1.0/reports/authenticationMethods/userRegistrationDetails (note: v1.0, not beta). Follow @odata.nextLink. Handle 403 with _permission_missing("AuditLog.Read.All"). Per RESEARCH.md Pattern 2 — do NOT use per-user authentication/methods iteration.

3. get_failed_signins_bulk(self, from_date: str, to_date: str) -> List[Dict[str, Any]]:
   Query /beta/auditLogs/signIns with $filter=status/errorCode ne 0 and createdDateTime ge {from_date} and createdDateTime le {to_date}, $orderby=createdDateTime desc, $top=500. Follow @odata.nextLink for pagination. Select: createdDateTime, userDisplayName, userPrincipalName, ipAddress, location, status, appDisplayName. Per D-07 hybrid approach.

**ReportSyncService** (app/services/report_sync_service.py) — new file:

Create class ReportSyncService(BaseConfigurableService) with config_prefix="reports".

Lazy-load graph_service and sku_catalog from container (use current_app.container.get pattern).

Methods:
- sync_license_data(self): Call graph_service.get_all_users_with_licenses() and graph_service.get_subscribed_skus(). Cross-reference to compute per-SKU summary: sku_name (friendly name from sku_catalog), prepaid_units.enabled (available), consumedUnits (consumed), assigned count from user iteration, unused_30d count (users with that SKU whose signInActivity.lastSignInDateTime is >30 days ago or null). Store two cache entries via ReportCache.store(): report_type="license_summary", cache_key="per_sku", ttl_hours=4, data=list of per-SKU dicts. Also store "license_summary"/"totals" with aggregate KPI values (total_skus, total_assigned, total_unused_30d, utilization_pct). Per REPT-01/REPT-02.

- sync_security_data(self): Call graph_service.get_mfa_registration_details(). Compute: total_users (filter userType eq member), mfa_registered count (isMfaRegistered==True), mfa_pct, list of users without MFA (userPrincipalName, displayName). Store as ReportCache "mfa_summary"/"totals" (ttl_hours=1) and "mfa_summary"/"users_without" (ttl_hours=1). Also call get_failed_signins_bulk with from_date = now - 72 hours, to_date = now. Store as "signin_failures"/"recent" (ttl_hours=1). Per REPT-03/REPT-04.

- get_failed_signins(self, window=None, from_date=None, to_date=None) -> dict: Hybrid query per D-07. If window in (24h, 72h, 7d, 30d): load from ReportCache "signin_failures"/"recent", filter in Python by window. If cache is stale or custom date range provided: call graph_service.get_failed_signins_bulk(from_date, to_date) directly. Return dict with entries list + metadata (source: "cache" or "live", count).

Apply @handle_service_errors(raise_errors=False) to sync methods.

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-08-02 | Elevation of Privilege | Report sync job endpoints | mitigate | @admin_or_portal_required decorator on all job trigger/status endpoints (existing pattern) |
| T-08-03 | Information Disclosure | Bulk user data in report_cache | accept | Admin-only access; data already visible in individual profile cards; no PII beyond what admins already see |
