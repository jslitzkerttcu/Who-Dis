**Main route function** — reports():
@require_role("admin"). Get tab from request.args.get("tab", "licenses"). If HX-Request header present, return _render_tab(tab). Otherwise render_template("admin/reports.html", active_tab=tab).

**Tab dispatcher** — _render_tab(tab: str):
Dict mapping: licenses -> _render_licenses_tab, security -> _render_security_tab. (Genesys and history tabs added in Plan 03.) Return 404 for unknown tabs.

**License tab** — _render_licenses_tab():
Query ReportCache.get_cached("license_summary", "totals") for KPI data and ReportCache.get_cached("license_summary", "per_sku") for table data. Pass both cache entries to template along with their is_stale and age_display. Render admin/partials/_report_licenses.html.

**Security tab** — _render_security_tab():
Query ReportCache.get_cached("mfa_summary", "totals") and ReportCache.get_cached("mfa_summary", "users_without") for MFA section.
For failed sign-ins: get window from request.args (default "72h"), from_date and to_date from request.args. Get report_sync_service from container, call get_failed_signins(window=window, from_date=from_date, to_date=to_date). Apply paginate() to sign-in entries for server-side pagination (page from request.args, per_page=25). Pass all data to admin/partials/_report_security.html.

**CSV exports:**

_csv_safe(value: str) -> str: Copy from job_role_compliance.py pattern (prefix =, +, -, @ with apostrophe).

export_license_csv():
@require_role("admin"). Query ReportCache.get_cached("license_summary", "per_sku"). If not found, abort(404). Build CSV with metadata rows: "Report: License Utilization", "Generated: {timestamp}", blank row. Headers: SKU Name, Assigned, Available, Consumed, Utilization %, Unused (30d). Data rows from cache.data. Return Response with text/csv mimetype and Content-Disposition attachment header per D-04.

export_security_csv():
@require_role("admin"). Two sections: MFA summary + failed sign-ins. Query both caches. Build CSV with metadata rows: "Report: Security Posture", "Generated: {timestamp}", blank row. MFA section: headers (Display Name, Email, MFA Registered), data from users_without cache. Blank row separator. Failed Sign-ins section: headers (User, Email, Timestamp, Failure Reason, IP Address), data from signin_failures/recent cache. Return CSV response.

**Sign-in date validation helper** — _validate_date(date_str: str) -> Optional[str]:
Try datetime.fromisoformat(date_str). Return ISO string on success, None on invalid input. Per T-08-04 threat mitigation.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin browser -> report routes | Admin-only; tab name and date params are user input |
| ReportCache.data -> Jinja2 templates | Cached JSON rendered in HTML templates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Tampering | CSV export data | mitigate | _csv_safe() prefixes dangerous characters (=, +, -, @) with apostrophe before writing to CSV |
| T-08-06 | Tampering | Date range filter params | mitigate | _validate_date() rejects non-ISO-8601 input before passing to Graph API OData filter |
| T-08-07 | Spoofing | Report route access | mitigate | @require_role("admin") on every route function; no public report endpoints |
| T-08-08 | Information Disclosure | Cached report data in templates | accept | Jinja2 autoescaping prevents XSS; data is admin-visible by design |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No new packages; all dependencies already in requirements.txt |
</threat_model>
