## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Flask routes | Form data and HTMX requests from admin browser |
| Flask routes -> WorkflowService | Validated form data passed to service methods |
| Templates -> User data | Employee names, skip reasons rendered in templates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Spoofing | workflow routes | mitigate | @require_role("admin") on all handlers in workflows.py |
| T-11-02 | Tampering | employee_name, skip_reason | mitigate | Jinja2 auto-escaping on all template output; SQLAlchemy parameterized queries |
| T-11-03 | Repudiation | item completion/skip/create | mitigate | audit_logger.log_admin_action() on every state change with g.user, IP, user_agent |
| T-11-04 | Tampering | CSRF on POST endpoints | mitigate | @csrf_double_submit.protect on complete_item, skip_item, cancel_workflow, create_workflow; hx-headers with X-CSRFToken in templates |
| T-11-05 | Information Disclosure | employee data in search results | mitigate | employee_search endpoint gated by @require_role("admin"); no sensitive data in typeahead results |
| T-11-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan |
</threat_model>
