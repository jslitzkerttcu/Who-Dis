# Phase 10 -- UI Design Contract

> Visual and interaction contract for REST API. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Phase 10 is primarily a backend phase. The single new admin UI surface is External API Token Management -- a new section inside the existing admin blueprint where an admin can create named bearer tokens, view the token list, and revoke individual tokens. The one-time token reveal pattern (display once at creation, never again) requires a dedicated modal-style reveal component. The OpenAPI docs page at `/api/v1/docs` is served by Swagger UI (a third-party static bundle) and has no WhoDis-branded chrome. All other admin UI elements (spacing, typography, color, component patterns) are inherited from the established WhoDis Tailwind system without modification.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| **"Create Token" button click** | Opens create token modal as a fixed overlay (`fixed inset-0 bg-gray-600 bg-opacity-50 z-50`). Modal centers vertically at `top-20`. Contains a single text input for token name. Confirm button disabled until name field has 2+ non-whitespace characters. Matches existing cache clear modal structure from `admin/_cache_actions.html:162-193`. |
| **Token name validation** | Real-time `input` event listener. Confirm button starts `opacity-50 cursor-not-allowed`. Enables at 2+ characters. Max length: 100 characters. |
| **Create token submit** | POST via HTMX to `/admin/api-tokens/create` with `hx-swap="none"`. Server responds with `HX-Trigger` containing the raw token string. Client JS receives the trigger, closes create modal, opens reveal modal populated with the token value. |
| **Reveal modal display** | Token string shown in a `font-mono text-sm bg-gray-100 rounded border p-3 break-all` block. Full token always visible -- no show/hide toggle (admin must read and copy immediately). Copy button alongside token. Warning banner (amber, `fa-exclamation-triangle`) above the token block. |
| **Copy token** | `navigator.clipboard.writeText(token)` -> `showToast('Token copied to clipboard', 'success', 3000)`. Copy button briefly shows `fa-check` for 2s then reverts. |
| **Reveal modal close** | Click "Done -- I've Copied the Token" button. Modal closes. Token list refreshes via HTMX (`hx-get` on the token list container with `hx-trigger="tokenCreated from:body"`). Token value is gone from memory -- cannot be recovered from UI. |
| **Revoke button click** | Opens revoke confirmation modal (standard `max-w-md`). Displays token name in modal body. No reason text field required (revocation is its own audit event). Confirm button is red. |
| **Revoke confirm** | DELETE or POST to `/admin/api-tokens/{token_id}/revoke` via HTMX. Server responds `HX-Trigger: tokenRevoked`. Token row updates in-place: Name goes `line-through text-gray-400`, status badge swaps to "Revoked", Revoke Token button disappears (row becomes read-only). Modal closes. Success toast appears. |
| **Cancel / Escape** | Closes any open modal. No network call. Matches existing pattern. |
| **Keyboard** | Modal traps focus. Tab cycles through fields and buttons. Escape closes modal. Confirm button not in tab order while disabled. |
| **Screen reader** | Create and revoke modals have `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to modal title. Token name input has `aria-required="true"`. |
| **Mobile (<=768px)** | Token table scrolls horizontally (`overflow-x-auto` wrapper). Modals go full-width with `mx-4` margin. "Last Used" and "Created" columns collapse to `text-xs`. |
| **API docs link** | `<a href="/api/v1/docs" target="_blank" rel="noopener">` with `fa-external-link-alt` icon. Positioned as a secondary link below or beside the section heading. No HTMX -- standard navigation to Swagger UI page. |
| **HTMX error handling** | Non-2xx from create/revoke endpoint: modal stays open, error message appears below form in `text-red-600 text-sm`. 401 follows existing redirect-to-login flow. 403 shows `Insufficient permissions. Admin role required.` |
| **Revoked token rows** | Kept in the list (not deleted from view). Shown with muted styling: name in `text-gray-400`, no actions column buttons. Provides audit context (who created, when revoked). Server does not return revoked tokens in API auth checks. |
