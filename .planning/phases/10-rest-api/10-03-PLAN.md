          - Call `token_service.create_token(name=name, created_by=g.user)`

          - Audit log: `audit_service.log_admin_action(user_email=g.user, action="api_token_revoked", target=token.name, details={"token_id": token.id})`
          - Return response with `HX-Trigger: tokenRevoked` to refresh token list
          - Include success toast message

       d. `api_token_list()` — GET `/admin/api-tokens/list` (HTMX partial refresh):

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin browser -> Token CRUD routes | Admin-only endpoints behind OIDC + @require_role("admin") |
| Raw token display -> Clipboard | One-time display; token cleared from DOM after modal close |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-11 | Information Disclosure | _token_reveal_modal.html | mitigate | Token cleared from DOM on modal close; never stored in localStorage/sessionStorage; shown once only |
| T-10-12 | Spoofing | admin api_tokens.py | mitigate | All routes gated by @require_role("admin"); CSRF protection via existing Flask-WTF middleware |
| T-10-13 | Repudiation | token create/revoke | mitigate | audit_service.log_admin_action called for every create and revoke with user email, action, details |
| T-10-14 | Information Disclosure | HX-Trigger header | accept | Raw token passes via HX-Trigger response header (HTTPS only in production; same-origin); acceptable for admin action over TLS |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan |
</threat_model>
